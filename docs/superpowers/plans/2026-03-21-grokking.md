# Implementation Plan: Grokking Phase Diagrams

## Tasks

### Task 1: Scaffold submission directory
Create `submissions/grokking/` with all required files: `__init__.py`, `conftest.py`, `requirements.txt`, empty `src/` and `tests/` directories.

### Task 2: Implement data generation (`src/data.py`)
- `generate_modular_addition_data(p=97)` -> returns all (a, b, (a+b)%p) triples
- `split_dataset(data, train_fraction, seed=42)` -> random train/test split
- Both functions are pure and easily testable

**Tests (`tests/test_data.py`):**
- Correct number of examples (p^2)
- All labels in [0, p-1]
- Split fractions are correct
- Deterministic with seed

### Task 3: Implement model (`src/model.py`)
- `GrokkingMLP(p, embed_dim, hidden_dim)` — nn.Module
  - Two Embedding layers (one per input), concatenate, Linear->ReLU->Linear->output(p)
  - `count_parameters()` method
- Must be < 100K params for all configs

**Tests (`tests/test_model.py`):**
- Output shape is (batch, p)
- Parameter count < 100K for all hidden_dim values
- Forward pass runs without error
- Deterministic with seed

### Task 4: Implement training loop (`src/train.py`)
- `train_model(model, train_data, test_data, config)` -> TrainResult
- TrainResult: dataclass with train_accs, test_accs, train_losses, test_losses, epoch_train_95, epoch_test_95
- Uses AdamW, cross-entropy, full-batch
- Logs metrics every 100 epochs
- Early stopping: if both train and test acc > 99% for 3 consecutive checks

**Tests (`tests/test_train.py`):**
- Training on trivial data converges
- TrainResult has expected fields
- Early stopping triggers correctly

### Task 5: Implement phase classification (`src/analysis.py`)
- `classify_phase(result: TrainResult)` -> Phase enum (CONFUSION, MEMORIZATION, GROKKING, COMPREHENSION)
- `compute_grokking_gap(result: TrainResult)` -> int or None
- `aggregate_results(sweep_results)` -> summary statistics

**Tests (`tests/test_analysis.py`):**
- Each phase correctly classified from synthetic TrainResult objects
- Grokking gap computed correctly
- Edge cases: threshold boundaries

### Task 6: Implement sweep orchestration (`src/sweep.py`)
- `run_sweep(weight_decays, dataset_fractions, hidden_dims, ...)` -> list of SweepResult
- SweepResult: config + TrainResult + phase classification
- Progress logging to stdout

### Task 7: Implement plots (`src/plots.py`)
- `plot_phase_diagram(results, hidden_dim, save_path)` — 2D heatmap (weight_decay x dataset_fraction)
- `plot_grokking_curves(results, save_path)` — example training curves for grokking runs
- Uses matplotlib only, `plt.savefig()` with `bbox_inches='tight'`

### Task 8: Implement report generation (`src/report.py`)
- `generate_report(sweep_results)` -> markdown string
- Sections: overview, phase counts, phase diagrams (image references), grokking gaps, key findings

### Task 9: Write `run.py` and `validate.py`
- `run.py`: orchestrates full pipeline, saves results to `results/`
- `validate.py`: checks that all expected outputs exist and are valid
