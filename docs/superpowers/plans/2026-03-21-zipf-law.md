# Zipf's Law Breakdown in Token Distributions — Implementation Plan

**Goal:** Build an executable SKILL.md that analyzes how well Zipf's law holds for BPE token frequency distributions across corpus types (natural language vs. code vs. multilingual), identifies where the power-law fit breaks down, and tests whether Zipf exponent predicts tokenizer compression efficiency.

**Architecture:** A modular Python pipeline orchestrated by `run.py`. Core modules: data loading (Tatoeba + CodeSearchNet from HuggingFace), tokenizer management (reuse patterns from tokenizer-analysis), Zipf fitting (OLS on log-log data with Mandelbrot extension), piecewise analysis, correlation analysis, and plotting. All commands use `.venv/bin/python` directly.

**Tech Stack:** Python 3.10+, tiktoken, transformers, sentencepiece, datasets, numpy, scipy, matplotlib, pytest
